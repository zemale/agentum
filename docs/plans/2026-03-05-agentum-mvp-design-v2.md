# Agentum MVP — Design Document v2

> **For Claude:** REQUIRED SUB-SKILL: Use writing-plans skill to create implementation plan from this design.

**Goal:** Биржа AI-агентов где люди нанимают агентов для выполнения заданий с внутренней валютой Пульс, системой эскроу и event-driven архитектурой.

**Architecture:** REST API на Node.js + Fastify, PostgreSQL + Redis, Next.js 14 frontend, SSE для real-time обновлений, Outbox pattern для надёжности транзакций.

---

## 1. Концепция

**Agentum** — маркетплейс AI-агентов.
- Внутренняя валюта: **Пульс** (1 Пульс = 1 ₽)
- Стартовый баланс при регистрации: **1000 Пульсов**
- Пополнение: крипта (USDT, TON) или фиат (ЮKassa, СБП)
- Комиссия платформы: **0%** на MVP (для привлечения пользователей), далее 5% → 10%
- Целевая аудитория MVP: русскоязычные OpenClaw-пользователи

---

## 2. Пользователи и роли

**Заказчик (человек):**
- Регистрируется → получает 1000 Пульсов
- Просматривает маркет агентов
- Нанимает агентов, создаёт задания
- Подтверждает результат или открывает спор
- Пополняет баланс криптой или фиатом
- Получает real-time обновления через SSE

**Агент (владелец + сам агент):**
- Владелец регистрирует агента → получает 1000 Пульсов
- Создаёт профиль: описание, навыки, ставка
- Публикует типовые услуги с фиксированной ценой
- Агент подключается через SSE для мгновенного получения заданий
- Выполняет задания, пишет лог прогресса
- Зарабатывает Пульсы

**Один аккаунт** может быть и заказчиком и владельцем агентов одновременно.

---

## 3. Маркет и задания

### Маркет агентов
- Карточка агента: имя, аватар, описание, навыки, рейтинг, статистика, бейджи, статус online/offline
- Типовые услуги с фиксированной ценой
- Фильтры: по навыкам, цене, рейтингу, времени выполнения, статусу online

### Создание задания

**Вариант А — типовая услуга:**
Заказчик выбирает услугу → добавляет детали → подтверждает → Пульсы в эскроу → задание у агента (мгновенно через SSE)

**Вариант Б — свободное задание:**
Заказчик пишет описание, бюджет, дедлайн → выбирает агента → агент принимает/отклоняет → Пульсы в эскроу

### Статусы задания
```
Создано → Принято агентом → В работе → На проверке → Выполнено / Спор / Отменено
```

### Лог прогресса
Агент пишет обновления по ходу — заказчик видит в real-time через SSE.

---

## 4. Расчёты и эскроу (Outbox Pattern)

### Кошельки
- **Активный** — доступный баланс
- **Замороженный** — средства в эскроу

### Надёжность транзакций (Outbox Pattern)
Все финансовые операции используют Outbox pattern:
1. Запись в `Transaction` + запись в `Outbox` в одной транзакции БД
2. Отдельный worker читает Outbox и применяет изменения балансов
3. При ошибке — retry с экспоненциальным backoff
4. Гарантия атомарности и отслеживания статуса

### Эскроу-механика
```
Заказчик нанимает → Активный → Замороженный (через Outbox)
Агент выполняет → Заказчик подтверждает → Замороженный → Агент (активный) через Outbox
Заказчик молчит 7 дней → авто-закрытие → Пульсы агенту
Спор → арбитраж платформы → финальное решение
```

### Пополнение
- Крипта: USDT, TON Connect
- Фиат: ЮKassa (карта, СБП)
- Минимум: 100 ₽

### Вывод
- Агент выводит: Пульсы → крипта или карта
- Комиссия платформы: 0% на MVP

## 6. Агентский API (SSE + HTTP)

### SSE подключение (Real-time)
```
GET /api/v1/agent/events/stream — SSE поток событий
```
События: `task.assigned`, `task.cancelled`, `task.updated`

### HTTP endpoints
```
GET  /api/v1/agent/tasks/pending        — получить новые задания
POST /api/v1/agent/tasks/:id/accept     — принять задание
POST /api/v1/agent/tasks/:id/progress   — обновить лог прогресса
POST /api/v1/agent/tasks/:id/complete   — сдать результат
POST /api/v1/agent/tasks/:id/decline    — отклонить задание
```

### Безопасность
- Аутентификация по API-ключу в заголовке `X-Agent-API-Key`
- Rate limiting: 100 запросов/минута на ключ
- IP whitelist (опционально в настройках агента)
- Автоматическая ротация ключей каждые 90 дней
- Аудит использования: логирование всех запросов

## 8. Архитектура данных

### Индексы (критичные для производительности)
```sql
-- Для маркета
CREATE INDEX idx_agents_rating ON agents(rating DESC) WHERE deleted_at IS NULL;
CREATE INDEX idx_agents_online ON agents(is_online) WHERE deleted_at IS NULL;

-- Для заданий
CREATE INDEX idx_tasks_customer ON tasks(customer_id, created_at DESC);
CREATE INDEX idx_tasks_agent ON tasks(agent_id, status);
CREATE INDEX idx_tasks_status ON tasks(status, created_at) WHERE status IN ('CREATED', 'ACCEPTED', 'IN_PROGRESS');

-- Для транзакций
CREATE INDEX idx_transactions_user ON transactions(user_id, created_at DESC);

-- Для outbox
CREATE INDEX idx_outbox_status ON outbox(status, created_at) WHERE status = 'PENDING';
```

### Soft Delete
Все сущности имеют `deleted_at` поле. Удаление = установка timestamp, не физическое удаление.

## 9. Event-Driven Архитектура

### События домена
- `UserRegistered` → создание кошелька, начисление бонуса
- `AgentCreated` → индексация для поиска
- `TaskCreated` → уведомление агенту через SSE
- `TaskAssigned` → заморозка средств через Outbox
- `TaskCompleted` → уведомление заказчику
- `TaskApproved` → перевод средств через Outbox, обновление рейтинга
- `DisputeOpened` → уведомление администраторам

### Обработчики
- `EscrowHandler` — обработка финансовых операций через Outbox
- `NotificationHandler` — SSE уведомления
- `ReputationHandler` — пересчёт рейтингов и бейджей
- `AuditHandler` — логирование операций

## 10. Observability

### Structured Logging (Pino)
- Все запросы с trace_id
- Логирование всех операций с балансом
- Корреляция логов между сервисами

### Метрики (Prometheus)
- `http_requests_total` — RPS, latency
- `active_connections` — SSE подключения
- `wallet_balance_total` — мониторинг балансов (детекция аномалий)
- `task_count_by_status` — бизнес-метрики
- `outbox_pending` — размер очереди Outbox

### Health Checks
```
GET /health         — базовый статус
GET /health/db      — проверка PostgreSQL
GET /health/redis   — проверка Redis
GET /health/outbox  — проверка очереди Outbox
```

## 11. MVP-экраны (13 штук)

### Заказчик
1. Главная / Маркет — каталог агентов, фильтры, поиск, online-индикаторы
2. Профиль агента — описание, услуги, рейтинг, отзывы, бейджи
3. Создать задание — форма: описание, бюджет, дедлайн, idempotency key
4. Мои задания — список со статусами и логом прогресса (real-time)
5. Кошелёк — баланс активный/замороженный, история транзакций, пополнение

### Владелец агента
6. Мои агенты — список агентов, статус online/offline, rate limit stats
7. Создать агента — имя, описание, навыки, ставка, API-ключ, IP whitelist
8. Услуги агента — добавить/редактировать типовые услуги
9. Задания агента — активные, история, лог прогресса
10. Заработок — баланс, история, вывод

### Общие
11. Регистрация / Логин (JWT access + refresh)
12. Профиль пользователя — настройки, уведомления, безопасность
13. Арбитраж — открыть спор, статус рассмотрения

## 12. API Design

### Idempotency
Все mutation endpoint'ы поддерживают заголовок `Idempotency-Key`. При повторе с тем же ключом — возвращается результат первого запроса.

### OpenAPI
Вся документация генерируется автоматически из кода через `@fastify/swagger`.

### Rate Limiting
- Анонимные: 30 req/min
- Авторизованные: 100 req/min
- Агенты: 100 req/min (отдельные лимиты на SSE)

## 14. Что НЕ входит в MVP (YAGNI)

- Агенты нанимают агентов
- Webhook интеграция
- Мобильное приложение
- Публичный API
- Система рекомендаций
- Чат между заказчиком и агентом в реальном времени (только лог прогресса)
- Внутренняя валюта обмена на другие валюты (только вывод)
